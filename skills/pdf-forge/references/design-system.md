# PDF Forge Design System Reference

This document defines the visual language for all PDFs generated by pdf-forge. Every decision here exists to produce output that looks designed by a human, not assembled by AI. Follow these specifications exactly.

---

## Typography

### Font Stack

Load Inter as a **variable font** from Google Fonts CDN. This enables fine-grained weight control (any value 100-900) instead of fixed weight steps. Fall back to `system-ui, -apple-system, sans-serif`. Use `font-mono` (system monospace) exclusively for section labels and category markers. Never mix more than two font families in a single document.

```html
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@100..900&display=swap" rel="stylesheet">
```

Apply `font-family: 'Inter', system-ui, -apple-system, sans-serif;` to the root element.

Variable font weights allow subtle hierarchy. Instead of jumping from 300 to 600, use intermediate values like 350 or 450 via `font-[350]` for nuanced contrast. Standard Tailwind weight classes remain valid but the variable font unlocks finer control when needed.

### Slide Scale (16:9 / 1920x1080)

Use these tokens when generating presentation slides rendered at 1920x1080 pixels.

| Token | Tailwind Classes | Usage |
|-------|-----------------|-------|
| Section Label | `text-xl font-mono uppercase tracking-label` | Categories, section markers, topic tags |
| Hero Heading | `text-7xl font-semibold tracking-heading leading-tight` | Primary slide title, one per slide |
| Sub Heading | `text-5xl font-semibold tracking-heading` | Secondary titles, slide subtitles |
| Card Title (L) | `text-4xl font-medium` | Hero cards in Bento grids, featured items |
| Card Title (S) | `text-2xl font-medium` | Smaller cards, supporting grid items |
| Body | `text-2xl text-zinc-400 font-light tracking-body leading-relaxed` | Descriptions, explanations, paragraphs |
| Caption | `text-xl text-zinc-500 font-light tracking-body` | Meta text, footnotes, attributions |
| Big Number | `text-8xl font-bold tracking-display` | Impact statistics, KPIs, metrics |
| Mega Number | `text-[120px] font-bold leading-none tracking-display` | ROI figures, hero values, dramatic stats |
| Price | `text-3xl font-semibold` | Monetary values, cost figures |

### Document Scale (A4 / 210x297mm)

Use these tokens when generating paginated documents at A4 dimensions (794x1123 pixels at 96 DPI).

| Token | Tailwind Classes | Usage |
|-------|-----------------|-------|
| Section Label | `text-xs font-mono uppercase tracking-label` | Categories, section markers |
| Heading 1 | `text-4xl font-semibold tracking-heading` | Page titles, major sections |
| Heading 2 | `text-2xl font-semibold tracking-heading` | Subsection headers |
| Heading 3 | `text-xl font-medium` | Minor headers, card titles |
| Body | `text-base text-zinc-400 tracking-body leading-relaxed` | Paragraph text, descriptions |
| Caption | `text-sm text-zinc-500 tracking-body` | Footnotes, metadata, attributions |
| Big Number | `text-5xl font-bold tracking-display` | Statistics, metrics |
| Price | `text-xl font-semibold` | Monetary values |

### Typography Principles

**All tracking is negative — calibrated from 5 production design systems.** The templates define four semantic tracking tokens via Tailwind config, derived from cross-referencing Figma, Stripe, Vercel, Framer, and Linear:

| Token | Value | Use for | Industry range |
|-------|-------|---------|---------------|
| `tracking-display` | -0.1em | Hero headings, mega numbers, big stats | -0.07 to -0.27em |
| `tracking-heading` | -0.06em | Standard headings, sub headings | -0.06em (unanimous) |
| `tracking-body` | -0.025em | Body text, descriptions, captions | -0.025 to -0.07em |
| `tracking-label` | -0.01em | Section labels, tags, mono text | -0.009 to -0.019em |

These tokens are defined in each template's `tailwind.config` block and replace generic Tailwind tracking classes. The semantic names communicate intent — `tracking-heading` is clearer than `tracking-[-0.06em]`.

**Never use `tracking-wide`, `tracking-wider`, or `tracking-widest`.** Positive letter-spacing is the single most recognizable marker of AI-generated PDFs. No production design system uses positive tracking — even the "widest" values in Figma (-0.009em), Stripe (-0.019em), and Framer (-0.009em) are still negative.

**Weight layering prevents visual fatigue.** Use `font-light` (300) for body text to keep large blocks of text from feeling heavy. Reserve `font-semibold` (600) for headings and `font-bold` (700) for numbers and emphasis. The weight differential between body and heading must be at least two steps (e.g., 300 to 600). With Inter's variable font, intermediate weights like `font-[350]` or `font-[450]` create more nuanced hierarchy.

**Size contrast must be aggressive.** Hero elements should be 3-4x the size of body text. A `text-7xl` heading next to `text-2xl` body text creates unmistakable hierarchy. Timid size differences (one step apart) produce ambiguous hierarchy and look unintentional.

## Color System

### Dark Theme (Default)

Dark backgrounds are the default for all generated PDFs. They produce higher perceived contrast, better visual weight for typography, and a more contemporary aesthetic.

| Role | Tailwind Class | Hex | Usage |
|------|---------------|-----|-------|
| Background | `bg-zinc-950` | #09090b | Page/slide background |
| Surface | `bg-zinc-900` | #18181b | Cards, panels, containers |
| Surface Elevated | `bg-zinc-800` | #27272a | Hover states, active cards |
| Border | `border-zinc-800` | #27272a | Card borders, dividers |
| Border Subtle | `border-zinc-800/50` | #27272a80 | Internal separators, soft dividers |
| Text Primary | `text-white` | #ffffff | Headings, primary content |
| Text Secondary | `text-zinc-400` | #a1a1aa | Body text, descriptions |
| Text Tertiary | `text-zinc-500` | #71717a | Captions, metadata |
| Accent Label | `text-orange-500` | #f97316 | Section labels, warm authority |
| Accent Border | `border-purple-500` | #a855f7 | Border-left highlights |
| Accent Gradient | `from-purple-400 to-orange-400` | — | Victory moments: ROI, hero stats |
| Glow | `bg-orange-600/10 blur-[100px]` | — | Decorative background glow |

### Light Theme (When Requested)

Only use light theme when explicitly specified. Map the palette accordingly:

| Role | Tailwind Class |
|------|---------------|
| Background | `bg-white` |
| Surface | `bg-zinc-50` |
| Surface Elevated | `bg-zinc-100` |
| Border | `border-zinc-200` |
| Border Subtle | `border-zinc-200/50` |
| Text Primary | `text-zinc-900` |
| Text Secondary | `text-zinc-600` |
| Text Tertiary | `text-zinc-400` |

### Color Principles

**Zinc backbone with contained accent.** The zinc scale provides all structural color. The accent system has three layers: `orange-500` for section labels, `purple-500` for border highlights, and a `from-purple-400 to-orange-400` gradient reserved exclusively for the highest-impact element per page (ROI, hero stat, key value). Never use the gradient on more than one element per slide. These defaults adapt when brand colors are configured.

**Opacity modulation for depth.** Use `/50`, `/10`, `/5` opacity suffixes to create layered surfaces without introducing new colors. Example: `bg-orange-600/10` creates a tinted glow that relates to the accent system without competing.

**Backgrounds define hierarchy.** `zinc-950` is the page. `zinc-900` is a card. `zinc-800` is emphasis within a card. This three-tier depth system is the only background layering needed.

## Spacing System

The spacing scale follows a **geometric progression** (inspired by Figma's design system), not linear multiples. Smaller values provide fine control for internal element spacing; larger values handle page-level structure.

### Spacing Scale

| Tier | Values (px) | Tailwind | Use for |
|------|-------------|----------|---------|
| Micro | 4, 8, 12 | `1`, `2`, `3` | Icon-to-text, inline gaps, badge padding |
| Small | 16, 24 | `4`, `6` | List items, card internal spacing, input padding |
| Medium | 32, 48 | `8`, `12` | Card padding, grid gaps, section sub-spacing |
| Large | 64, 80, 96 | `16`, `20`, `24` | Page padding, header-to-content, major sections |
| XL | 128, 160 | `32`, `40` | Dramatic vertical rhythm, hero spacing |

Only values from this scale are permitted. Arbitrary pixel values are prohibited.

### Slide Spacing (1920x1080)

| Context | Value | Tailwind | Notes |
|---------|-------|----------|-------|
| Page padding | 96px | `p-24` | All four sides of the slide |
| Header to content | 64-80px | `mb-16` to `mb-20` | Space below title block |
| Grid gap | 24-48px | `gap-6` to `gap-12` | Between grid items |
| Card inner padding | 32-40px | `p-8` to `p-10` | Content within cards |
| Internal separator | 24px | `pb-6 border-b border-zinc-800/50` | Dividers within cards |
| Between major blocks | 64px | `space-y-16` | Vertical rhythm between sections |
| Icon to text | 16-24px | `gap-4` to `gap-6` | Icon-label pairs |

### Document Spacing (A4)

| Context | Value | Tailwind | Notes |
|---------|-------|----------|-------|
| Page padding | 48-64px | `p-12` to `p-16` | Page margins |
| Section spacing | 32-48px | `mb-8` to `mb-12` | Between major sections |
| Paragraph spacing | 16-24px | `mb-4` to `mb-6` | Between paragraphs |
| Card padding | 24-32px | `p-6` to `p-8` | Card interiors |
| Table cell padding | 12-16px | `p-3` to `p-4` | Table cells |
| List item spacing | 8-12px | `space-y-2` to `space-y-3` | Vertical list items |

### Spacing Principles

**Content-to-whitespace ratio:** target 60:40 for slides, 70:30 for documents. When in doubt, add more whitespace. Fewer items with breathing room always beat more items crammed together.

**Generous margins over cramped content.** If content does not fit comfortably with `p-24` slide padding, reduce the content -- never reduce the padding. Remove items, shorten text, or simplify the layout.

**Vertical rhythm must be consistent.** Pick one major spacing value (e.g., `space-y-16`) and use it between all top-level blocks on a slide. Inconsistent vertical gaps make a layout feel accidental.

**Never use arbitrary spacing values.** Every measurement must come from the geometric scale. Valid Tailwind spacing utilities: `1`, `2`, `3`, `4`, `6`, `8`, `10`, `12`, `16`, `20`, `24`, `32`, `40`.

## Layout Principles

### Visual Hierarchy

**One focal point per slide or page.** Every composition has exactly one element that dominates. If a slide has a hero stat, that stat is the focal point -- everything else supports it. If everything competes for attention, nothing wins.

**Size contrast establishes hierarchy.** The focal element should be 3-4x larger than supporting elements. A `text-8xl` number next to `text-2xl` body text is unambiguous. A `text-4xl` number next to `text-2xl` body text is weak.

**Proximity communicates relationship.** Group related items with tight spacing (`gap-4`). Separate unrelated groups with wide spacing (`gap-12` or more). The gap ratio between related and unrelated items should be at least 3:1.

### Composition Patterns

#### Bento Grid

Asymmetric card grid where one hero card spans 2 columns or 2 rows, with smaller cards filling the remaining space. This creates natural hierarchy without explicit visual emphasis.

```html
<div class="grid grid-cols-3 gap-6">
  <div class="col-span-2 row-span-2 bg-zinc-900 rounded-2xl p-10">
    <!-- Hero card -->
  </div>
  <div class="bg-zinc-900 rounded-2xl p-8">
    <!-- Supporting card -->
  </div>
  <div class="bg-zinc-900 rounded-2xl p-8">
    <!-- Supporting card -->
  </div>
</div>
```

Use Bento when presenting 3-6 related items where one item deserves emphasis.

#### Split Screen

50/50 horizontal divide for creating mental contrast. Left side holds analytical/structural content (stats, lists, specs). Right side holds emotional/visual content (hero number, illustration placeholder, testimonial).

```html
<div class="grid grid-cols-2 gap-12">
  <div><!-- Analytical content --></div>
  <div><!-- Impactful content --></div>
</div>
```

Use Split Screen for before/after, problem/solution, or data/insight pairings.

#### Stat Block with Border Accent

Vertical stat groups anchored by a `border-l-4` accent line. Creates structure without heavy visual elements.

```html
<div class="border-l-4 border-emerald-400 pl-6">
  <div class="text-8xl font-bold text-white">94%</div>
  <div class="text-2xl text-zinc-400 font-light">Customer retention rate</div>
</div>
```

Use for KPIs, metrics, and statistics that need to stand alone.

#### Stacked Sections

Full-width blocks separated by consistent vertical spacing. Each block contains a section label, heading, and content area.

```html
<div class="space-y-16">
  <section>
    <div class="text-xl font-mono uppercase tracking-label text-zinc-500 mb-4">Section Label</div>
    <h2 class="text-5xl font-semibold tracking-heading text-white mb-6">Section Title</h2>
    <p class="text-2xl text-zinc-400 font-light leading-relaxed max-w-3xl">Content here.</p>
  </section>
</div>
```

Use for linear narratives, agendas, and document-style pages.

### Component Patterns

#### Cards

Always use `rounded-2xl` for slide-scale cards and `rounded-xl` for document-scale cards. Apply `bg-zinc-900` as the default card surface. Never use shadows on dark backgrounds -- they are invisible. Use border (`border border-zinc-800`) only when cards sit adjacent to each other and need visual separation.

#### Tables

Style tables with alternating row backgrounds for readability:

```html
<table class="w-full text-left">
  <thead>
    <tr class="border-b border-zinc-800">
      <th class="pb-4 text-sm font-mono uppercase tracking-label text-zinc-500">Header</th>
    </tr>
  </thead>
  <tbody>
    <tr class="border-b border-zinc-800/50">
      <td class="py-4 text-base text-zinc-300">Value</td>
    </tr>
  </tbody>
</table>
```

#### Badges and Tags

Small metadata indicators: `inline-flex items-center px-3 py-1 rounded-full text-xs font-medium bg-emerald-500/10 text-emerald-400`. Keep badge text short (1-2 words). Never use more than 3 badges in a row.

#### Dividers

Use `border-b border-zinc-800/50` for internal card divisions. Use `border-b border-zinc-800` for major section divisions. Never use `<hr>` -- always use border utilities on a container element.

### Text Width Constraints

Constrain body text width to maintain readable line lengths:
- Slides: `max-w-3xl` to `max-w-4xl` (48-56rem)
- Documents: `max-w-prose` (65ch) or `max-w-2xl` (42rem)

Never let body text run the full width of a slide. Long lines degrade readability and look undesigned.

### What NOT to Do

**Equal-sized grids for everything.** A 3x3 grid of identical cards has zero hierarchy. Use Bento or vary card sizes.

**Center-aligning all text.** Center alignment works for single lines (titles on cover slides). For multi-line content, always left-align. Mixed alignment within a card is forbidden.

**Edge-to-edge content.** Content touching the slide/page edges looks suffocating. The `p-24` slide padding is non-negotiable.

**Bullet points as default.** When presenting 3-5 items, use cards or spatial grouping instead of bullets. Bullet lists are acceptable only inside document-scale body text or when items exceed 6.

**Decorative borders on every element.** Borders are structural, not decorative. Only add a border when two adjacent surfaces share the same background color and need visual separation.

**Drop shadows on dark backgrounds.** Shadows are invisible on `zinc-950`. Use surface color differentiation (`zinc-900` vs `zinc-800`) instead.

**Gradients as backgrounds.** Subtle gradients on individual elements are acceptable (`bg-gradient-to-br from-zinc-900 to-zinc-800`). Full-page gradients are prohibited -- they compete with content.

## Page Structure

### Slide HTML Shell

Every slide is a standalone HTML page at exactly 1920x1080 pixels:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=1920">
  <script src="https://cdn.tailwindcss.com"></script>
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@100..900&display=swap" rel="stylesheet">
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { font-family: 'Inter', system-ui, -apple-system, sans-serif; }
  </style>
</head>
<body>
  <div class="w-[1920px] h-[1080px] bg-zinc-950 text-white p-24 flex flex-col">
    <!-- Slide content -->
  </div>
</body>
</html>
```

### Document HTML Shell

A4 documents use 794x1123px pages (210x297mm at 96 DPI):

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <script src="https://cdn.tailwindcss.com"></script>
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@100..900&display=swap" rel="stylesheet">
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { font-family: 'Inter', system-ui, -apple-system, sans-serif; }
    .page { width: 794px; min-height: 1123px; }
    @media print { .page { page-break-after: always; } }
  </style>
</head>
<body>
  <div class="page bg-zinc-950 text-white p-16">
    <!-- Page content -->
  </div>
</body>
</html>
```

### Multi-Page Documents

For documents with multiple pages, stack `.page` divs vertically. Each page is an independent composition -- content must not rely on overflow between pages. Manually break content across pages to ensure no element is cut mid-render.

## Quality Checklist

Before finalizing any generated PDF, verify:

1. **Typography hierarchy** -- at least 3 distinct size levels are visible on each slide/page.
2. **Tracking tokens** -- `tracking-label` (-0.01em), `tracking-body` (-0.025em), `tracking-heading` (-0.06em), `tracking-display` (-0.1em). All negative, calibrated from 5 production systems.
3. **Weight layering** -- body is `font-light`, headings are `font-semibold` or `font-bold`.
4. **Whitespace ratio** -- content does not exceed 60% of slide area or 70% of document area.
5. **Single focal point** -- one element clearly dominates per slide/page.
6. **Color discipline** -- only zinc scale plus one accent color in use.
7. **Text width** -- body text is constrained with `max-w-*`, not running full width.
8. **No visual anti-patterns** -- no equal grids, no center-aligned paragraphs, no edge-to-edge content, no shadows on dark backgrounds.
